Underline links on hover with pseudo-elements

Look at what happens when you hover over this word.

Here is the code that was used to make this effect :

a::after {
content: "";
border-bottom: 4px solid blue;
position: absolute;
top: 19px;
left: 0;
right: 0;
bottom: 0;
transform: scale(0,1);
transition: all 200ms ease-in-out;
}
a:hover::after {
transform: scale(1, 1);
}

Reset best practise

In modern browsers, the reset below can be used to reset box sizing, margins and paddings :

html {
box-sizing: border-box;
}
*,
*::before,
*::after {
margin: 0;
padding: 0;
box-sizing: inherit;
}

Adjacent sibling combinators

Sometimes, setting a margin-right to all items causes the last button on the right to also have a margin to the right. To prevent this, write this instead of the margin-right :

.nav__item + .nav__item {
margin-left: 1em;
}

Centering with margins

It is possible to use blocks to center margins. Be careful though: the block you're working on needs to have a set width to have a set width in order for this to work.

.content {
margin: 0 auto;
}

Sources

Most of the content on this site were things i learned through Kevin Powell's Conquering Responsive Layouts course. Some other tips gathered from his course are :

  • Use "%" for widths. Default for block elements is 100% of parent element.
  • Set a max-width so content doesn't stretch too much on larger screens.
  • For fonts, use rem, not em as em will always look at parent element and font sizes will get bigger the farther they are nested in their parent element.
  • For properties such as margins, paddings, widths or any other property than font-size, using em will make said property look at the font-size of the element.
  • When possible, use class names instead of element names.
